13 June 2026
Rank trend sparkline in /profile
Data
/profile now shows a rank progression chart built from historical snapshots stored locally in the bot's SQLite database.
  • Every rank refresh writes a new snapshot to a rank_history table — deduplication ensures only actual rank changes are recorded, not noisy re-checks.
  • The chart renders as a Unicode block-character sparkline (▁▂▃▄▅▆▇█) with the earliest and latest rank and the time span shown below it.
  • History builds up passively from today — no backfill required.
/match current renamed to /match latest — now public
Commands UX
The /match current subcommand has been renamed to /match latest and is now a public (non-ephemeral) response so players can share their most recent game in chat.
  • /match history remains public as before.
Prototype page — visual flows for Riot API review
UX
Added /prototype — a public page with visual flows and Discord embed mockups of the bot's core features, built to support the Riot API and RSO application.
  • Covers the full account verification flow, all four player commands, the live rank distribution chart, and staff moderation tools.
JRMA deployment recovery via Docker push
Infra
JRMA's git-based build pipeline went down ("No matching nodes") preventing the bot from being updated via the normal deploy flow.
  • Resolved by building the Docker image locally and pushing it directly to JRMA's container registry, bypassing their broken build nodes.
  • Bot is back online running the latest code. Normal npm run ship deploys resume as usual once JRMA's nodes recover.
Ephemeral responses for private commands
UX
Commands with personal or sensitive output are now ephemeral — only visible to the user who ran them, keeping channels clean.
  • Ephemeral: /profile, /match latest, /link.
  • Public (shareable): /match history, /leaderboard.
  • Always public: /admin link panel and /admin link bulk-reset.
Site redesign — neo-brutalist aesthetic
UX
Full redesign of the public landing page with a neo-brutalist / neo-industrial visual language — hard offset shadows, dot-grid texture, thick borders, and uppercase monospace labels throughout.
  • Bot profile card now uses the real Discord avatar and banner pulled directly from CDN, with the bot bio displayed beneath the avatar.
  • Hard red offset box-shadows (8px 8px 0 0) on the bot card; ghost offset border behind it for depth.
  • Dot-grid background texture via radial gradient at 22 px spacing.
  • Stats strip reduced to three columns — removed the static "51k+ Server Members" figure in favour of the live Discord count shown in the community section.
Rank distribution chart — tracker.gg-style breakdown
Data
The landing page now shows a live rank distribution chart for all linked server members, from Iron 1 through Radiant.
  • Bot generates stats on startup and every 6 hours via src/utils/generate-stats.js, writing to the Redis key stats:rank-distribution.
  • New Vercel serverless endpoint /api/stats reads the Redis key — no direct DB access from the frontend.
  • 25 bars rendered (Iron 1 → Radiant), each coloured by rank group, with percentage labels and hover tooltips showing exact counts.
  • Chart displays total linked, ranked, and unranked counts plus a human-readable "Updated X ago" timestamp.
Fix: /match filtering out invalid game modes
Commands UX
Deathmatch, Swift Play, and other modes the API returns no player data for were showing as "Unknown · 0/0/0 · 0 ACS" in match history and latest.
  • /match history now fetches 15 matches and filters to the first 10 with real data before displaying.
  • /match latest fetches the 5 most recent matches and skips past any with missing player or map data.
Fix: /leaderboard timing out on large servers
Commands
The leaderboard was fetching all 51,000+ guild members before responding, exceeding Discord's interaction window.
  • Removed the guild member fetch entirely — linked accounts are queried directly from the database instead, making the command near-instant.
12 June 2026
/match — Last match & match history
Commands UX
Two new subcommands for match data, with a single API call each.
  • /match latest — detailed last match view with full team & enemy breakdown, agent, K/D/A, ACS, and HS%.
  • /match history — your last 10 matches at a glance with per-match result, agent, K/D/A, ACS, and an overall W/L record.
  • Supports an optional user mention or direct riot_id argument for looking up any player.
/leaderboard — Server rank leaderboard
Commands
Paginated leaderboard of all linked server members, sorted by rank tier then RR.
  • 10 members per page with Previous / Next buttons.
  • Embed colour automatically reflects the highest rank on the current page.
  • Immortal+ members display their live leaderboard position (#N) where available.
  • Fix: removed an expensive full guild member fetch that was causing the command to time out on large servers.
Background account validation — 6-hour scans
Security
Every 6 hours the bot re-checks every linked account against the user's current Discord connections.
  • Auto-invalidates the link and removes rank roles if the Riot connection has been removed or swapped to a different account.
  • Silently updates stored name/tag if the PUUID matches but the display name changed.
  • Posts a scan summary to the staff log channel with pass / renamed / invalidated / error counts.
Account swap detection on Update Rank
Security UX
When a user hits the Update Rank button, the bot now verifies their current Discord connections in real time before fetching rank.
  • Warns the user if their Riot connection has been removed or points to a different account.
  • Shows a live “Checking your account…” → “Fetching your rank…” animation during the check.
  • Adds a verified badge to the success embed when the connection is confirmed.
Admin commands hidden from non-administrators
Security Admin
/admin and its subcommands are now hidden at the Discord level from anyone without the Administrator permission — they won't appear in the command list at all.
  • Server admins can grant access to specific roles via Server Settings → Integrations → [Bot] without any redeploy.
/export moved to /admin export
Admin UX
The standalone /export command has been folded into /admin export, keeping admin tooling consolidated under a single command group.
OAuth token storage for connection re-checking
Data Security
Discord OAuth access and refresh tokens are now stored at link time, enabling the bot to re-query a user's Discord connections on demand without asking them to re-authorise.
  • Tokens are refreshed automatically when they expire.
  • Used by both the Update Rank flow and the background 6-hour validation scan.
Unified deploy script — npm run ship
Infra
A single npm run ship command now pushes to GitHub, JRMA (bot server), and Vercel in sequence — no more manual three-step deploys.
Public landing page with live member count
Infra UX
Launched valorant-oce-utilities.vercel.app with a Valorant-themed landing page.
  • Member count updates live on page load using the Discord invite API — no rounding.
  • Privacy Policy and Terms of Service pages added.
